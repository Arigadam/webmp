# p2p-overlay

Мультиплеер поверх любой веб-страницы — без сервера. Подключает посетителей сайта друг к другу через WebRTC (сигналинг через BitTorrent DHT, [Trystero](https://trystero.dev)).

## Возможности

- **Живые курсоры** — видите мышь / касание каждого посетителя в реальном времени
- **Чат** — общий для всего домена, сообщения видят все вкладки сайта
- **Системный лог** — кто зашёл, ушёл, перешёл на другую страницу
- **Счётчик присутствия** — «сейчас на этой странице» и «сейчас на сайте»
- **Список людей** — вкладки «На странице» и «На сайте» с именами и путями
- **Своё имя** — случайное при первом входе, меняется кликом по таблетке «вы: …»
- **Мобильная поддержка** — тачи отображаются так же, как курсоры на ПК
- **SPA-роутинг** — отслеживает `pushState` / `replaceState` / `popstate`
- **Без сервера** — нет бэкенда, нет базы данных, нет WebSocket-сервера

## Подключение

Один тег перед `</body>` — больше ничего не нужно:

```html
<script src="p2p-overlay.js"></script>
```

или если вы пользователь [установите UserScript](https://arigadam.github.io/webmp/webmp.user.js).

Trystero подгружается автоматически изнутри скрипта через dynamic import с CDN.

## Как это работает

```
Браузер A  ──WebRTC──  Браузер B
     │                      │
     └──── BitTorrent DHT ──┘
           (сигналинг, без своего сервера)
```

Каждый посетитель сайта вступает в две «комнаты» Trystero:

| Комната | ID | Что передаётся |
|---|---|---|
| Сайт | `site` | чат, приветствия, смена имени, навигация |
| Страница | `page:<origin+pathname>` | курсоры / тачи |

Комнаты изолированы по домену через `appId: 'p2p-overlay:' + location.hostname`.

## Структура файлов

```
p2p-overlay.js   — весь скрипт, один файл, без зависимостей при сборке
test.html        — тестовая страница 1
page2.html       — тестовая страница 2
```

## Конфигурация

В начале `p2p-overlay.js` есть объект `CONFIG`:

```js
const CONFIG = {
  appId:      'p2p-overlay:' + location.hostname, // изоляция сети по домену
  siteRoomId: 'site',
  pageRoomId: 'page:' + location.origin + location.pathname,
  maxLogLines: 80,   // максимум строк в логе чата
  palette: [...],    // цвета курсоров / аватаров
};
```

## Требования

- Современный браузер с поддержкой WebRTC и ES-модулей (Chrome 80+, Firefox 75+, Safari 14+)
- Интернет-соединение (для начального сигналинга через DHT)
